How the Model Works
01
Static Analysis
Lizard scans every file for cyclomatic complexity, function count, and lines of code — without running the code.
02
Git History Mining
PyDriller extracts commit frequency, churn, author patterns, and recent activity from the full git log.
03
Bug Labeling (SZZ)
Commits with keywords like "fix" or "bug" are identified. The previous commit touching those lines is labeled buggy.
04
XGBoost Prediction
A gradient-boosted model trained on cross-project data outputs a 0–1 risk probability per file.
05
SHAP Explainability
SHAP values decompose each prediction into per-feature contributions so you know exactly why a file is flagged.
06
Actionable Output
Risk scores, ranked file lists, and plain-English explanations help teams prioritize reviews and testing.

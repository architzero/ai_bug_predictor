What is CodeSentinel?

Software bugs are expensive. Most bugs cluster in a small fraction of files. CodeSentinel uses machine learning trained on real open-source projects to predict which files in your codebase are most likely to contain bugs, so you can focus review and testing where it matters.

How it works

Languages Supported

Python JavaScript TypeScript Java Go Ruby PHP C# C++ Rust

Limitations

  • Requires git history (bare file upload not supported).
  • Labels derived from SZZ have an estimated ~30% noise rate (standard in academic literature).
  • Model generalizes best to codebases similar in style to the training repos.
  • Bug type prediction is approximate (macro F1 ~0.65) — treat as a hint.

Training Data

The model is trained on a diverse set of open-source projects across multiple languages and domains to ensure generalizability.

What the risk score means

"A score of 78% does not mean 78% certainty of a bug. It means this file ranks in the highest risk tier based on all signals combined. The model is calibrated: its average predicted probability matches the actual bug rate in the training data (17% in both cases), so it is honest rather than overconfident."
